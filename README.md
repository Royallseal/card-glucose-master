# 《卡牌控糖师》— Card Glucose Master

《卡牌控糖师（Card Glucose Master）》是一款将 **Roguelike 牌组构筑** 与 **糖尿病医学知识** 深度结合的单人卡牌对战游戏。玩家扮演架空世界中的"卡牌控糖师"，通过打出膳食卡、运动卡、药物卡等各类卡牌，在精确管理自身血糖的同时击败具象化的代谢病变敌人，最终净化两个世界。

## 三、 核心玩法

### 1. 战斗循环

每回合抽取 5 张手牌，拥有 3 点能量用于打出卡牌。卡牌可造成伤害、提供格挡或调整血糖。玩家初始生命值为 80，归零则游戏失败。击败全部敌人即可通关。

### 2. 血糖博弈

血糖值在 2.0 ~ 15.0 mmol/L 范围内波动，低于下限或超出上限均会直接导致死亡。血糖所处的区间会全局修正卡牌表现，形成高收益与高风险并存的博弈：

| 血糖区间 | 状态 | 伤害 / 格挡修正 | 血糖波动 |
| :--- | :--- | :---: | :---: |
| 4.4 ~ 7.0 | **健康** | 提升 25% | 正常 |
| 2.0 ~ 4.3 | 低血糖 | 无修正 | 正常 |
| 7.1 ~ 15.0 | 高血糖 | 削减 25% | **翻倍** |

### 3. 卡牌体系

卡牌分为四大类，覆盖攻、防、控糖三个维度：

- **膳食卡**（6 张）—— 通常升糖，附带攻击或格挡效果。
- **运动卡**（6 张）—— 通常降糖，附带伤害或状态施加。
- **药物卡**（6 张）—— 强力降糖或特效干预，用于紧急控糖。
- **初始卡**（2 张）—— 米饭（膳食）与散步（运动），开局自带。

击败敌人后可在三张随机卡牌中选取一张加入牌组，并有机会在商店中购买高品质卡牌或删除冗余卡牌。

### 4. 状态系统

战斗中可通过卡牌对自身或敌人施加以下状态（Buff / Debuff）：

| 状态名称 | 类型 | 效果 | 消耗方式 |
| :--- | :---: | :--- | :---: |
| **活力** | 属性增益 | 每层攻击伤害 +1 | 永久 |
| **耐力** | 属性增益 | 每层格挡值 +1 | 永久 |
| **脆弱** | 减益 | 受击伤害提升 50% | 回合 |
| **乏力** | 减益 | 输出伤害降低 25% | 回合 |
| **僵硬** | 减益 | 获得格挡降低 25% | 回合 |
| **缓释** | 次数盾 | 抵消下一次血糖上升 | 次数 |
| **敏化** | 次数盾 | 使下一次降糖效果翻倍 | 次数 |

## 四、 数据管线

本项目采用 **CSV 母表 → 编辑器导表 → JSON 运行时加载** 的数据驱动管线，确保数值策划的绝对调整权：

1. 策划在 Excel 中维护数值母表（`data/initial_cards_data.csv` 等）。
2. 运行 Python 审计脚本 `scripts/audit_and_visualize.py` 校验卡牌配平，并生成可视化分析图表。
3. Unity Editor 启动时，`CSVImporter` 自动将 CSV 转换至 `Assets/Resources/Configs/` 下的 JSON 配置文件。
4. 游戏运行时，`CardDatabase` 与 `EnemyDatabase` 读取 JSON 并实例化数据实体。

> [!NOTE]
> 所有战斗数值（伤害、格挡、血糖变化等）**严禁在 C# 代码中硬编码**，必须通过上述管线统一配置。
