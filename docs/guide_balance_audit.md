# 《卡牌控糖师》数值配平审计指南

本指南定义《卡牌控糖师》卡牌数值的自动化审计与可视化流程。通过运行独立的 Python 脚本，对卡牌平衡性的各个维度进行量化监控。

---

## 一、 运行环境配置

为避免依赖版本冲突，配平审计工具须在独立的 Python 虚拟环境（.venv）中运行。

### 1. 初始化与激活虚拟环境 (Windows)
于项目根目录下执行以下命令创建并激活虚拟环境：

```powershell
# 创建虚拟环境（生成 .venv 文件夹）
python -m venv .venv

# 激活虚拟环境 (Windows PowerShell)
.venv\Scripts\Activate.ps1

# 激活虚拟环境 (Windows CMD)
.venv\Scripts\activate.bat
```

### 2. 安装依赖包
在激活的虚拟环境中，执行以下命令安装绘图相关的第三方依赖库：

```bash
pip install -r scripts/requirements.txt
```

---

## 二、 执行审计脚本

激活虚拟环境后，在项目根目录下运行审计脚本：

```bash
python scripts/audit_and_visualize.py
```

脚本执行后自动完成以下工作：
1. 解析卡牌数据源 `data/initial_cards_data.csv`。
2. 依据天平公式校验单卡数值，于终端输出《数值平衡配平审计账单》与《战斗属性平均能耗统计》。
3. 渲染 7 张独立的平衡与分布分析图表，并保存至 [docs/assets/](./assets) 目录。

---

## 三、 数值审计图表解读

生成的 7 张分析图表涵盖卡牌平衡审计的各个维度：

### 1. 能量消耗 vs 血糖变化散点图
![能量与血糖变化散点图](assets/card_scatter_energy_vs_glucose.png)
* **读图说明**：
  * **横轴/纵轴**：横轴代表血糖变化值（负值为降糖，正值为升糖），纵轴代表能量消耗（0-2费）。
  * **散点尺寸**：尺寸与卡牌的理论价值点数（Target VP）成正比。
  * **散点颜色**：对应卡牌稀有度（灰色-初始，蓝色-普通，橙色-良好，紫色-优秀）。
  * **重合处理**：坐标重合的多张卡牌应用径向发散算法（Jitter）错开显示。图表不含文字标注以确保界面整洁。
* **分析目的**：监控卡牌的血糖与费用空间分布，防止高升糖卡牌堆积在低费区或大额降糖卡牌堆积在高费区。

### 2. 战斗属性平均能耗柱状图
![战斗属性平均能耗](assets/card_bar_avg_cost.png)
* **读图说明**：展示伤害卡（Attack）、格挡卡（Defense）与功能卡（Utility）的平均能量消耗。
* **分析目的**：监控各类卡牌的均费曲线以确保战斗节奏流畅。目前伤害卡均费控制在 1.17 费，格挡卡均费控制在 0.83 费，功能卡均费控制在 1.00 费，整体能耗结构保持均衡。

### 3. 卡牌价值偏差审计图
![卡牌价值偏差审计](assets/card_bar_vp_deviation.png)
* **读图说明**：展示各单卡实际分配价值与天平公式折算价值的偏差值（Deviation）。
  * **绿色条（偏差 $\le \pm0.3$）**：数值配平符合预期。
  * **蓝色条（偏差 $<-0.3$）**：数值分配保守或故意削弱。初始卡【米饭】与【散步】特意设计为 -1.0 VP 偏差，以拉开稀有度阶梯。
  * **红色条（偏差 $>0.3$）**：数值分配超出预算（超模），属于重点审计对象。
* **分析目的**：快速锁定数值失衡或设计超模的卡牌。

### 4. 卡牌伤害输出分布图
![卡牌实际伤害输出分布](assets/card_bar_damage.png)
* **读图说明**：展示所有伤害类卡牌的单次总伤害额度（多段攻击已按总段数折算）。
* **分析目的**：确认伤害卡牌的输出梯度是否合理，确保伤害值按品质与费用呈现递增阶梯。

### 5. 卡牌格挡值分布图
![卡牌防御格挡值分布](assets/card_bar_block.png)
* **读图说明**：展示所有格挡类卡牌的格挡值分布。
* **分析目的**：审计生存保障梯度，防止低费防守卡提供过高盾量导致生存压力失真。

### 6. 卡牌战斗属性纯化占比饼图
![卡牌战斗属性纯化占比](assets/card_pie_attribute_composition.png)
* **读图说明**：展示纯伤害卡、纯格挡卡与功能卡在卡组中的构成比例。
* **分析目的**：落实“无攻防混载”的卡牌纯化设计要求，目前伤害卡占 30%、格挡卡占 30%、功能卡占 40%，符合预设的卡组结构配比。

### 7. 各卡牌血糖升降波动影响图
![各卡牌血糖升降波动影响](assets/card_bar_glucose_impact.png)
* **读图说明**：卡牌按血糖变化值降序排列。正值为升糖（红色柱体），负值为降糖（绿色/蓝色柱体）。
* **分析目的**：监控单卡瞬时血糖负荷，审计升糖或降糖幅度是否偏离常规数值界限。

## 四、 数据导出与版本提交规范

1. **数据导出**：若在数值母表中调整了卡牌属性，须导出为 CSV 格式覆盖 `data/initial_cards_data.csv`。
2. **提交流程前置校验**：提交至 Git 仓库前，须在激活的虚拟环境中运行 `scripts/audit_and_visualize.py`，确保不存在非预期超模（红色偏差条）。
3. **Git 提交规范**：图表更新后，生成的 7 张 PNG 图表须与 `data/initial_cards_data.csv` 一同提交。Git 提交日志前缀使用 `design:`。
